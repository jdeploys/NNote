# Project Instructions

## Test execution scope

- 기능·버그 수정 중에는 변경 동작을 직접 검증하는 최소 테스트만 실행한다.
- 공통 계약, DB 스키마, 녹음·복구, 아카이브처럼 영향 반경이 넓을 때만 관련 스위트로 확장한다.
- 전체 단위·통합·E2E·시각·패키징 검증을 일상적인 완료 절차로 실행하지 않는다.
- `npm run test:release`는 릴리스·태그·배포 준비 또는 사용자의 명시적 요청에만 실행한다.
- 새 테스트는 기존 테스트가 잡지 못하는 고유한 실패를 설명할 수 있어야 한다. 같은 결과를 단위·E2E·시각 테스트에서 반복 검증하지 않는다.
- 문서 스크린샷 생성은 회귀 테스트와 분리한다.
